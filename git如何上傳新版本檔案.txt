如何將您的專案變更推送到 GitHub
這份指南將引導您完成將本地專案的最新變更推送到您已建立的 GitHub 儲存庫（repository）的完整流程。

前提
請確認您已經：

在您的電腦上安裝了 Git。

您的專案資料夾已經是一個 Git 儲存庫（也就是您之前已經執行過 git init 或從 GitHub clone 下來）。

您的本地儲存庫已經與遠端的 GitHub 儲存庫建立連結。

步驟 1：打開您的終端機 (Terminal) 或命令提示字元 (Command Prompt)
首先，您需要打開一個命令列工具。

Windows: 您可以使用 Command Prompt、PowerShell 或 Git Bash。

macOS / Linux: 您可以使用 Terminal。

接著，使用 cd 指令切換到您的專案資料夾目錄。

# 範例：將路徑換成您自己的專案路徑
cd /path/to/your/project

步驟 2：檢查目前的變更狀態
在執行任何操作前，最好先查看一下目前有哪些檔案被修改了。這是一個好習慣。

git status

這個指令會顯示出哪些檔案被修改、哪些是新增的、哪些已經被放到「暫存區」(staging area)。

步驟 3：將變更加入暫存區
在提交（commit）您的變更之前，您需要先將想要包含在這次提交中的所有檔案加入「暫存區」。

如果您想加入所有變更的檔案（最常見的做法）：

git add .

（. 代表目前目錄下的所有變更）

如果您只想加入特定的檔案：

# 將 '檔案名稱' 換成您想加入的檔名
git add 檔案名稱

步驟 4：提交變更
將檔案加入暫存區後，您需要「提交」這些變更。提交就像是為您的專案建立一個新的存檔點，並且需要附上一段訊息來描述您這次做了什麼修改。

# 將 "您的提交訊息" 換成一段有意義的描述
git commit -m "您的提交訊息"

好的提交訊息範例：

git commit -m "Feat: 新增課程分頁功能"

git commit -m "Fix: 修正行動版跑版問題"

git commit -m "Docs: 更新專案說明文件"

步驟 5：推送到 GitHub
最後一步就是將您剛剛建立的提交推送到 GitHub 遠端儲存庫。

git push

如果您是第一次推送，或是您的本地分支名稱與遠端不同，您可能需要使用更完整的指令，指定要推送到哪個遠端儲存庫的哪個分支：

# 通常遠端儲存庫的預設名稱是 origin，主要分支是 main
git push origin main

提示：您的主要分支名稱可能是 main 或 master，您可以透過 git branch 指令來查看。

執行完 git push 後，終端機會顯示推送進度。完成後，您就可以到您的 GitHub 網站上查看，您的最新變更已經成功上傳了！
